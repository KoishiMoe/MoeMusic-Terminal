# MoeMusic Terminal

[简体中文](./README_zh.md) | English

> [!WARNING]
> This project is experimental and is not recommended for production environments. All code was generated by AI and has only been manually tested. No code audit has been conducted (other than checking for unknown third-party dependencies). Security vulnerabilities may exist when processing untrusted audio or image files.

MoeMusic Terminal is a standalone, lightweight text terminal client for the [MoeMusic](https://github.com/lolicode-org/MoeMusic) runtime. 

Although the core library of MoeMusic is highly abstract and platform-independent, it was designed primarily for the `MoeMusic-Minecraft` mod. This sub-project serves as a standalone proof of concept (POC) to run completely outside of Minecraft, designed to satisfy my local playback needs. Because I do not have UI design expertise, the interface remains spartan.

## Features

- **TUI Interface**: Powered by Lanterna.
- **Config Management**: Configuration is generated at `moemusic.toml` and must be edited externally.
- **Java Sound Playback**: Uses Java Sound instead of OpenAL for audio output, eliminating complex native library dependencies.
- **Single-Process Bridge**: The server and client communicate through an in-memory packet bridge, keeping the standard shared request and playback protocol intact.
- **Plugin Support**: Drop custom audio source or plugin JARs under `<config-dir>/plugins/` to extend capabilities.

## Installation & Running

### Run directly with Gradle
To run the project in development mode:
```bash
./gradlew run
```

### Linux / macOS
For a proper text terminal, do not use `gradle run` because Gradle may launch the JVM without a controlling terminal (`/dev/tty`), which Lanterna requires on Unix-like systems. Instead, install the distribution and run the generated script:
```bash
./gradlew installDist
./build/install/moemusic-terminal/bin/moemusic-terminal
```

### Windows
On Windows, use the generated batch file launcher from Windows Terminal, PowerShell, or Command Prompt to ensure the JLine backend attaches properly to the active console:
```powershell
gradlew.bat installDist
build\install\moemusic-terminal\bin\moemusic-terminal.bat
```

### Android
The included native libraries have been built for Android and successfully run in the Termux environment on Android devices. Follow the Linux/macOS installation instructions.

Termux does not currently support the Sixel image protocol as of this writing (2026-05-25), so cover display will fall back to Unicode half-block rendering. 
The good news is that the Termux development team has prepared a [patch](https://github.com/termux/termux-app/pull/2973) to add native Sixel support, which should be merged in the next release. 
At that time, you can enable Sixel image rendering with the `--cover sixel` parameter for improved cover display quality.

## Command Line Arguments

- `--config-dir <path>`: Specifies the configuration directory. Defaults to the OS-specific user config directory.
- `--terminal <mode>`: Selects terminal mode explicitly (`auto`, `text`, or `swing`).
- `--mouse <mode>`: Mouse control settings (`auto`, `on`, or `off`). Mouse support defaults to `auto` and depends on the terminal emulator emitting mouse events.
- `--cover <mode>`: Cover art rendering settings (`auto`, `terminal`, `kitty`, `sixel`, `unicode`, or `off`).
  - Cover display defaults to terminal images on known Kitty-protocol or Sixel-capable terminals. Other terminals fall back to Unicode half-block rendering.
  - Use `--cover kitty` to force Kitty graphics, `--cover sixel` to force Sixel graphics, `--cover terminal` for conservative image terminal detection, `--cover unicode` for text rendering, or `--cover off` to disable cover art entirely.
  - Covers will fall back to an inline placeholder if the file is missing, blocked by the client's media firewall, or fails to decode.

## Known Issues

- **Konsole Image Blurriness**: When running under Konsole on Linux, cover art rendering may appear blurry. Due to the developer's technical limitations, this issue cannot currently be resolved (it is unclear whether this is a Konsole-specific limitation). Cover art renders correctly in Kitty.
- **Windows Terminal image position and sizing**: In Windows Terminal, cover images may be misaligned or rendered at incorrect sizes. After resizing the window, a placeholder colored area may remain outside the image bounds.

## Technical Details

- **Runtime Logs**: Logs are written to `<config-dir>/terminal.log` while the TUI is active, preventing background warnings or logs from corrupting the terminal display.
- **Java 24+ Compatibility**: The generated launcher enables native access for unnamed modules. This suppresses the restricted native-access warning emitted by LavaPlayer before application logging is redirected away from the terminal.
